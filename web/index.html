Send a message to a grpc cluster. If say 'rust' is chosen as target cluster and there is a delegate configured for the receiving endpoint(implemented in java ) - the call will be handled by the delegate. In case the delegate fails or not configured - call will be handled by the receiving endpoint(implemented in rust) itself. Communication between the backend endpoints is over grpc - while browser to envoy proxy is over grpc-web.

Delegates are configured while starting the endpoints in a stand-alone setup or via configmap - if deployed on to a kubernetes cluster.
